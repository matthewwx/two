## 一、产品概述

**一句话**：公司机房一站式运维管理系统平台，覆盖服务器、网络设备、容器集群的全生命周期管理。一条命令部署，运维能用、领导能看。

### 核心原则

- **不造轮子**：拼装业界最佳开源工具，零软件成本
- **一键部署**：`docker compose up -d` 全栈拉起
- **开箱即用**：别人拿过去也能直接跑
- **平滑扩展**：Docker Compose 起步，Helm Chart 预留 K8s 迁移路径
- **双重视角**：运维技术面板 + 领导展示大屏，同一套数据源

### 实现方式

- **路由控制**：Nginx 反向代理根据分组放行不同路由
- **双因素认证**：支持 TOTP / WebAuthn

## 三、管理范围

| 类别 | 设备举例 | 管理方式 |
|------|---------|---------|
| 🖥️ **服务器** | 物理机、虚拟机 | Prometheus + node_exporter + Cockpit |
| 🌐 **网络设备** | 路由器、交换机、防火墙、WAF | SNMP 发现 + LibreNMS 监控 |
| 🐳 **容器/K8s** | Docker、K8s Pod | cAdvisor + kube-state-metrics |
| 📦 **资产** | 全部设备和软件许可证 | NocoDB CMDB 统一记录 |

### 网络设备支持

**协议**：SNMP v1/v2c/v3、ICMP Ping、SSH、Telnet

**品牌覆盖**：Cisco、Huawei、H3C、Juniper、Fortinet、Palo Alto、以及任何支持 SNMP 的网络设备

**新增流程**：新设备开启 SNMP → LibreNMS 自动发现 → 自动归类 → Grafana 面板自动更新

## 四、核心组件与选型理由

| 组件 | 用途 | 为什么选它 |
|------|------|-----------|
| **Nginx** | 统一入口 + HTTPS 反向代理 | 成熟稳定，Let's Encrypt 自动证书 |
| **LibreNMS** | 网络发现 + SNMP 监控 + 设备告警 | 原生支持网络设备，CDP/LLDP 拓扑发现 |
| **Prometheus** | 主机/容器深度指标采集 | 业界标准，K8s 原生集成 |
| **Grafana** | 统一可视化 + 报表生成 | 支持多数据源，内置告警面板和报表插件 |
| **NocoDB** | CMDB 资产管理 | 在线表格，非技术人员也能维护 |
| **Cockpit** | 远程终端 + 文件管理 + 服务操作 | 浏览器里 SSH，零客户端 |
| **n8n** | 自动化工作流引擎 | 告警 → 自愈 → 通知全自动，300+ 集成 |
| **Loki + Promtail** | 日志集中管理 | 比 ELK 轻量 10 倍，Grafana 原生集成 |
| **Authentik** | 统一认证 + 权限 + 审计 | 开源自建，支持 OIDC/OAuth2/SAML |
| **OpenVAS** | 漏洞扫描 + 合规检查 | 开源漏洞库最全，定期自动扫描 |
| **Vaultwarden** | 密码管理 | 团队共享密码不传明文，全平台客户端 |
| **restic + MinIO** | 配置备份 | 增量加密去重，S3 兼容存储 |
| **Dashy / Homepage** | 门户首页 | 配置 YAML 即可搭建，支持双模切换 |

## 五、网络拓扑自动生成

```
SNMP/CDP/LLDP 邻居发现 → LibreNMS → 设备关系图 → 实时拓扑
```

### 能力清单

| 功能 | 说明 |
|------|------|
| 🔍 自动发现 | CDP/LLDP 邻居信息 + ARP 表 + FDB 表 + 路由表 |
| 🎨 实时着色 | 🟢 在线 → 🟡 告警 → 🔴 离线 |
| 🖱️ 可交互 | 点设备弹出详情（IP、型号、端口流量） |
| ⚠️ 故障定位 | 链路断开自动标红线 |
| 📥 导出 | PNG/SVG 格式，用于文档或汇报 |

### 实现方案

| 阶段 | 方案 | 工具 |
|------|------|------|
| **首选**（第一阶段） | LibreNMS 内置拓扑 | LibreNMS CDP/LLDP 发现 |
| **增强**（第四阶段） | 美观拓扑图 | LibreNMS 数据 + draw.io MCP 自动生成 |

### 技术实现

```
Grafana（数据源）
    ├── Grafana Reports 插件 → PDF 自动生成
    ├── Grafana API → n8n 工作流 → Excel 模板填充
    └── n8n 数据采集 → Document Generator agent → PPT 排版

下载：门户首页「报表中心」→ 按月份筛选 → 一键下载
```

### 报表结构示例

- 第 4 页：安全摘要 — 扫描漏洞数 / 已修复 / 待处理

**月报 PPT（6 页）**：
- 第 1 页：封面
- 第 2 页：本月核心指标（红黄绿大卡片）
- 第 3-4 页：趋势图（可用率 / 容量 / 安全）

- 第 6 页：下月重点

## 八、日志与审计

### 设计原则

> **所有操作可追溯，所有事件可搜索，定期出审计报告。**

### 八个数据源，统一汇入

```
┌─────────────────────────────────────────────────────────────┐
│                      日志与审计体系                           │
│                                                              │
│  数据源                        采集方式       存储/查询       │
│  ─────────────────────────────────────────────────────────  │
│  🖥️ 系统日志 (syslog/auth.log)   promtail  →                 │
│  🐳 Docker 日志                  Loki driver →               │
│  🌐 Nginx 访问/错误日志          file log →    Loki          │
│  ⌨️ 命令历史 (bash history)      promtail  →    +            │
│  🎥 Cockpit 会话录像             cockpit  →    Grafana       │
│  👤 Authentik 登录/登出          API  →        查询面板      │
│  📋 NocoDB 数据变更              webhook →                   │
│  🚨 告警处理记录                 n8n  →                      │
│                                                              │
│  输出：Grafana 审计面板 + 定期审计报告（PDF）                 │
└─────────────────────────────────────────────────────────────┘
```

### 日志采集明细

| 日志类型 | 来源 | 采集工具 | 保留期 |
|------|------|------|:---:|
| 🖥️ 系统日志 | syslog, auth.log, dmesg | promtail | 90 天 |
| 🐳 容器日志 | Docker stdout/stderr | Loki Docker Driver | 30 天 |
| 🌐 访问日志 | Nginx access/error | promtail file scrape | 90 天 |
| ⌨️ 命令历史 | `~/.bash_history` 所有用户 | promtail + 文件标签 | 180 天 |
| 📦 审计日志 | auditd（敏感文件变更） | promtail | 180 天 |

### 审计功能

#### 1. Cockpit 会话录像

```
管理员/用户通过 Cockpit 打开终端
    ↓
Cockpit Session Recording（内置功能）
    ↓
回放录像：ASCII cast 格式，浏览器直接播放
    ↓
存入 Loki 索引：会话时间、用户、目标机器、持续时间
```

| 能力 | 说明 |
|------|------|
| 🎥 自动录像 | 登录即录制，无感知 |
| 🔍 可搜索 | 按用户/时间/目标机器搜索 |
| ▶️ 可回放 | 浏览器内回放完整会话 |
| 🔐 权限 | 管理员可看所有人，普通用户看自己 |

#### 2. 命令历史集中管理

```
每台机器 bash_history → promtail 定时采集 → Loki
    ↓
Grafana 面板：按用户/机器/时间范围搜索
    ↓
示例查询：
  "谁在 .177 上执行了 rm -rf"
  "过去 7 天所有 sudo 操作"
  "所有机器上执行了 docker exec 的记录"
```

#### 3. CMDB 变更日志

| 记录内容 | 说明 |
|------|------|
| 👤 谁改的 | Authentik 用户 ID |
| 🕐 什么时候 | 精确到秒 |
| 📝 改了什么 | 字段级别：旧值 → 新值 |
| 📋 关联设备 | 影响哪台机器 |

NocoDB webhook → n8n → Loki 写入，Grafana 面板查询。

#### 4. 告警处理记录

```
告警触发 → n8n 通知运维 → 运维确认/处理
    ↓
记录：告警内容、触发时间、确认人、处理方式、处理耗时
    ↓
审计用途：哪些告警响应慢、谁处理了、有没有遗漏
```

#### 5. 操作审计面板（Grafana）

| 面板 | 内容 | 受众 |
|------|------|------|
| 📊 运维操作总览 | 今日登录次数、活跃会话、命令执行量 TOP 用户 | 管理员 |
| 🔍 命令搜索 | 全文搜索所有机器命令历史 | 管理员 |
| 🎥 会话回放 | Cockpit 录像列表 + 回放入口 | 管理员 |
| 📋 变更追踪 | NocoDB CMDB 变更记录 + 告警处理时间线 | 管理员 |
| 🔒 异常检测 | 非工作时间登录、敏感命令告警、多次失败登录 | 管理员 |

#### 6. 审计报告

| 报告 | 频率 | 格式 | 内容 |
|------|------|------|------|
| 📋 操作审计周报 | 每周一 | PDF | 本周登录统计、敏感操作清单、异常事件 |
| 🔒 安全审计月报 | 每月 1 号 | PDF | 登录失败趋势、权限变更、会话录像抽查、命令风险评级 |

## 十一、部署架构

### 当前方案：Docker Compose（单机）

```
.171 CentOS
├── Docker Engine
│   ├── Nginx (443/80)             ← 统一入口 + HTTPS
│   ├── LibreNMS (8000)            ← 网络发现 + SNMP
│   ├── Grafana (3000)             ← 统一可视化
│   ├── Prometheus (9090)          ← 指标采集
│   ├── NocoDB (8081)              ← CMDB
│   ├── Cockpit (9091)             ← 远程管理（每台被管机也装 agent）
│   ├── n8n (5678)                 ← 自动化工作流
│   ├── Loki (3100)                ← 日志聚合
│   ├── Authentik (9000)           ← 统一认证
│   ├── OpenVAS (9392)             ← 漏洞扫描
│   ├── Vaultwarden (8082)         ← 密码管理
│   ├── MinIO (9001)               ← 备份存储
│   ├── restic（无端口，定时执行）   ← 增量备份
│   └── Dashy/Homepage (8080)      ← 门户首页
│
├── Nginx 反向代理 → 统一域名 + Let's Encrypt 自动续期
└── restic → MinIO 每天凌晨 3 点备份
```

### 被管设备标准配置

```
每台被管设备（服务器 / VM / 网络设备）：
├── node_exporter       → Prometheus 指标
├── promtail            → Loki 日志采集
├── snmpd               → LibreNMS 发现
├── Cockpit agent       → 远程管理
└── Nginx 反向代理密码   → 流量加密
```

### 未来扩展：K8s Helm Chart

```
deploy/
├── docker-compose/          ← 现在直接用
│   ├── docker-compose.yml
│   └── .env
│
└── helm/                    ← 将来 K8s 迁移
    ├── grafana/
    │   ├── Chart.yaml
    │   ├── values.yaml
    │   └── templates/
    ├── prometheus/
    ├── librenms/
    └── ... 每个组件一个 Chart

迁移命令：
  docker compose down
  helm install ops-platform ./helm/
  # 数据通过 PersistentVolume 保留，不丢失
```

### 🟢 第一阶段：核心监控（第 1 周）

| 任务 | 工具 | 产出 |
|------|------|------|
| 自动发现 + 网络设备监控 | LibreNMS | 所有设备入网 |
