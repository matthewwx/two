# 同类开源项目对比分析

> 为 [[IT运维管理系统平台]] 找 6 个同类开源项目，分析差异和借鉴价值。

---

## 二、逐项目详细分析

### 1. RuoYi-Vue3-FastAPI（若依 FastAPI 版）

**技术栈**：FastAPI + Vue3 + Element Plus + SQLAlchemy + MySQL/PostgreSQL + Redis

**内置功能（17项）**：
- 用户/角色/菜单/部门/岗位管理
- 字典管理、参数管理、通知公告
- 操作日志、登录日志、在线用户监控
- 定时任务、服务监控（CPU/内存/磁盘）、缓存监控
- **代码生成器**（一键生成前后端 Python/SQL/Vue/JS 代码）
- 系统接口自动文档

**🟢 比 OpsBrain 强的地方**：
- 代码生成器：数据库表 → 一键生成全套 CRUD 代码
- 服务监控：CPU/内存/磁盘实时监控（OpsBrain 没有）
- 部门/岗位管理：更完善的组织架构
- 定时任务：内置 Celery/APScheduler 任务调度
- 在线用户：实时查看谁在线
- 菜单管理：动态菜单配置

**🔴 不如 OpsBrain 的地方**：
- 没有漏洞生命周期管理
- 没有 CMDB 资产代理
- 没有 OpenVAS 集成
- 没有 Grafana/Prometheus 嵌入
- 没有知识库 Markdown 文档

**💡 借鉴建议**：
- ✅ 直接复用代码生成器（省 80% CRUD 代码）
- ✅ 复用服务监控模块（CPU/内存/磁盘）
- ✅ 复用定时任务调度
- ❌ 不替换 — 它只是架子，没有运维业务

---

### 2. FastSoyAdmin

**技术栈**：FastAPI + Vue3 + Naive UI + Tortoise ORM + Redis + Docker

**亮点**：
- CLI 代码生成器（`cli-crud` 从模型一键生成前后端）
- 三层 RBAC + 行级数据范围控制
- **Radar 监控面板**（请求/SQL/异常/权限拒绝监控）
- fastapi-guard 限流 + IP 封禁
- 业务模块自动发现 + 多库支持
- AI Coding 友好（内置 CLAUDE.md）

**🟢 比 OpsBrain 强的地方**：
- CLI 代码生成器（更高效）
- Radar 监控面板（API 请求/SQL 性能实时监控）
- 限流 + IP 封禁（安全防护）
- 行级数据权限（OpsBrain 只有功能级 RBAC）

**🔴 不如 OpsBrain 的地方**：
- 完全没有任何运维业务功能
- 没有漏洞/工单/知识库/资产
- 没有外部工具集成

**💡 借鉴建议**：
- ✅ 复用 Radar 监控面板（直接加到仪表盘）
- ✅ 复用 CLI 代码生成器思路
- ✅ 借鉴行级数据权限设计
- ❌ 不替换 — 纯架子，且 ORM 不同（Tortoise vs SQLModel）

---

### 3. veops/cmdb（⭐ 最值得借鉴的 CMDB）

**技术栈**：Python + Vue.js + Ant Design Vue + MySQL + Redis

**核心功能**：
- 资源自动发现（计算机、网络设备、数据库、中间件、公有云）
- 多维度视图（资源/层级/关系视图）
- 自定义模型（灵活配置资产属性）
- 权限管控 + API 集成
- Docker Compose 一键部署

**🟢 比 OpsBrain 强的地方**：
- 资源自动发现（比我们的手动添加强太多）
- 自定义模型（用户可以自己定义资产类型和字段）
- 关系视图（设备之间的依赖关系图）
- 层级视图（机房→机柜→服务器层级）

**🔴 不如 OpsBrain 的地方**：
- 没有漏洞管理
- 没有工单系统
- 没有知识库
- 没有告警聚合
- 没有工具集成

**💡 借鉴建议**：
- ✅ 直接复用它的资源自动发现逻辑
- ✅ 借鉴自定义模型设计（让资产类型可配置）
- ✅ 借鉴关系/层级视图（当前 OpsBrain 只是平铺表格）
- ❌ 不替换 — 它只是 CMDB，不是运维平台

---

### 4. Vul-Manager（⭐ 最直接的对标项目）

**技术栈**：FastAPI + shadcn/ui + SQLAlchemy + PostgreSQL/SQLite + pandas + AI

**核心功能**：
- 漏洞列表（多维筛选 + 分页）
- 漏洞详情（CVSS 向量、受影响产品、修复建议、检测逻辑）
- Excel 漏洞工单导入
- **AI 智能风险分析**（支持 OpenAI/DeepSeek/通义千问/豆包）
- 仪表盘（总量/严重度/修复率/超期统计）
- 趋势图 + Top CVE + 导出 Excel

**🟢 比 OpsBrain 强的地方**：
- AI 风险分析（最亮眼的差异化功能）
- Excel 拖拽导入（比我们的粘贴框更友好）
- CVSS 向量解析（更专业的漏洞详情展示）
- 修复率/超期统计（更细粒度的仪表盘）

**🔴 不如 OpsBrain 的地方**：
- 没有资产 CMDB
- 没有任务工单
- 没有知识库
- 没有告警聚合
- 没有外部工具集成
- 没有 RBAC 权限（只有基础登录）

**💡 借鉴建议**：
- ✅ **最重要：直接借鉴 AI 漏洞分析功能**
- ✅ 借鉴 CVSS 向量解析
- ✅ 借鉴 Excel 拖拽导入
- ✅ 借鉴修复率/超期仪表盘
- ❌ 不替换 — 功能太单一，只是漏洞管理

---

### 5. open-cmdb（经典国产 CMDB）

**技术栈**：Django + Django REST Framework + Vue2 + iView

**核心功能**：
- Dashboard 仪表盘
- 机房/机柜/服务器管理
- 业务线与项目管理
- **Web SSH 终端**（浏览器里直接 SSH 到服务器）
- 自动数据采集
- 报表展示 + 定时任务

**🟢 比 OpsBrain 强的地方**：
- Web SSH 终端（非常实用！）
- 机房/机柜/服务器层级管理
- 更成熟的资产采集

**🔴 不如 OpsBrain 的地方**：
- Django 技术栈（不是 FastAPI）
- Vue2 + iView 已过时
- 没有漏洞/工单/知识库
- 没有告警/监控集成

**💡 借鉴建议**：
- ✅ **借鉴 Web SSH 终端功能**（用 ttyd 或 xterm.js 实现）
- ✅ 借鉴机房/机柜层级管理思路
- ❌ 不替换 — 技术栈陈旧

---

### 6. Industrace

**技术栈**：FastAPI + Vue3 + Vite + SQLAlchemy + PostgreSQL

**核心功能**：
- Dashboard 总览
- 资产管理 + 网络拓扑可视化
- 风险评估
- 多租户架构
- RBAC 权限控制
- 全局搜索 + 变更追踪

**🟢 比 OpsBrain 强的地方**：
- 网络拓扑可视化（设备关系图）
- 多租户架构（如果有多个团队）
- 更完善的风险评估

**🔴 不如 OpsBrain 的地方**：
- 面向工业 ICS/OT，不面向通用 IT
- 没有漏洞管理
- 没有工单/知识库
- 没有告警/监控

**💡 借鉴建议**：
- ✅ 借鉴网络拓扑可视化
- ✅ 借鉴多租户设计思路
- ❌ 不替换 — 行业不匹配

## 四、结论与建议

### 🏆 无需替换

**OpsBrain 是唯一同时覆盖「漏洞 + 工单 + 知识库 + CMDB + 监控集成」的运维平台。** 6 个竞品各有侧重，但没有一个能做到全功能覆盖。

### 📥 最值得借鉴的功能（按优先级）

| 优先级 | 借鉴来源 | 功能 | 实现难度 |
|--------|---------|------|---------|
| **P0** | Vul-Manager | AI 漏洞分析（接 DeepSeek） | 2天 |
| **P0** | veops/cmdb | 资源自动发现（nmap + SNMP） | 3天 |
| **P1** | RuoYi-FastAPI | 代码生成器（省 80% CRUD 工作） | 3天 |
| **P1** | RuoYi-FastAPI | 服务监控（CPU/内存/磁盘） | 1天 |
| **P1** | open-cmdb | Web SSH 终端（xterm.js） | 1天 |
| **P2** | FastSoyAdmin | Radar 监控面板 | 1天 |
| **P2** | Industrace | 网络拓扑可视化 | 2天 |
| **P2** | Vul-Manager | Excel 拖拽导入漏洞 | 1天 |

### 🚫 不推荐直接替换的原因

1. **若依/FastSoyAdmin** 只是空壳子，没有运维业务逻辑，替换=重新写一遍
2. **veops/open-cmdb** 只是 CMDB，缺漏洞/工单/知识库，替换后功能倒退
3. **Vul-Manager** 只有漏洞管理，功能太单一
4. **Industrace** 面向工业场景，通用性不足

**当前 OpsBrain 的「胶水层」定位是最优解**——底层开源工具继续用，自研只做聚合和补缺。后续优先把 P0/P1 功能加进来即可。
