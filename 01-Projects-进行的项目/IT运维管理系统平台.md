# IT 运维管理系统平台

> 基于开源工具的低代码运维管理系统平台，覆盖资产管理、监控告警、远程管理、安全扫描全流程

架构图见 [[01-Projects-进行的项目/IT运维管理系统平台五层架构图]]

## 核心理念

不造轮子，用业界最好的开源工具拼装。所有组件 Docker 化部署，统一在 Nginx 反向代理后面。

## 二、12 大需求对应方案

### 🔍 1. 自动发现 — LibreNMS

| 项目 | 说明 |
|------|------|
| 工具 | LibreNMS |
| 部署 | Docker Compose |
| 功能 | 输入 `192.168.3.0/24` → 自动扫描所有设备 → 识别 OS/服务/SNMP |
| 周期 | 每 6 小时自动扫描，新设备自动加入 |

### 📊 2. 监控 — Grafana + Prometheus + LibreNMS

| 组件 | 用途 |
|------|------|
| LibreNMS | 网络设备监控、流量、端口状态 |
| Prometheus + Node Exporter | 服务器深度指标（CPU/内存/磁盘/进程） |
| Grafana | 统一仪表盘，所有数据一个页面看 |

### 🚨 3. 告警 — LibreNMS Alerting + Webhook

| 场景 | 动作 |
|------|------|
| 设备离线 > 5 分钟 | 钉钉/企业微信通知 |
| CPU > 90% 持续 10 分钟 | 告警 + n8n 自动处理 |
| 磁盘 > 85% | 提前预警 |
| 服务端口不通 | 立即通知 |

### 📋 4. CMDB 资产管理 — NocoDB

```
在线表格，记录：
┌─────────┬──────────┬────────┬──────┬─────────┬──────────┐
│ 设备名   │ IP        │ 类型    │ 机房 │ 负责人  │ 到期日    │
├─────────┼──────────┼────────┼──────┼─────────┼──────────┤
│ Kali    │ .177      │ 物理机  │ 自建 │ 一号    │ -        │
│ ECShop  │ .168      │ 云主机  │ -   │ 一号    │ 2027-01  │
│ ...     │ ...       │ ...    │ ...  │ ...    │ ...      │
└─────────┴──────────┴────────┴──────┴─────────┴──────────┘
```

### 💻 5. 远程管理 — Cockpit

| 功能 | 说明 |
|------|------|
| Web 终端 | 浏览器里直接 SSH，不用再开终端 |
| 文件管理 | 上传/下载/编辑文件 |
| 服务管理 | 启停 systemd 服务 |
| 日志查看 | 实时查看系统日志 |
| 安装 | 每台被管机器 `apt install cockpit` |

### 📝 6. 日志管理 — Loki + Grafana

| 组件 | 用途 |
|------|------|
| Promtail | 每台机器装一个，收集 syslog/auth.log/应用日志 |
| Loki | 集中存储，轻量级，不像 ELK 那么吃资源 |
| Grafana | 同平台查询，不切换工具 |

### 🔐 7. 安全扫描 — OpenVAS (Greenbone)

| 功能 | 说明 |
|------|------|
| 漏洞扫描 | 定期扫所有设备，出漏洞报告 |
| 合规检查 | CIS Benchmark 基线检查 |
| 周期 | 每周自动扫描一次 |
| 集成 | 结果推送到 Grafana 面板展示 |

### 🔄 8. 自动化 — n8n

```
示例工作流：
  服务器 CPU > 90% → 钉钉通知 → 等 5 分钟
    → 还是高 → SSH 重启服务 → 通知结果
    → 恢复了 → 记录日志 → 结束

  每周五 18:00 → 生成周报 → 发邮件给领导
```

### 📈 9. 报表 — Grafana Reports + n8n

| 报表 | 内容 | 频率 |
|------|------|------|
| 日报 | 告警汇总 + 异常设备 | 自动 |
| 周报 | 可用率、容量趋势、变更记录 | 每周一 |
| 月报 | 安全扫描结果、SLA、成本 | 每月 1 号 |

### 👥 10. 权限管理 — Authentik

| 功能 | 说明 |
|------|------|
| 统一登录 | 一个账号登录所有系统 |
| 角色权限 | 管理员/运维/只读 |
| 操作审计 | 谁在什么时候做了什么 |
| 双因素 | 支持 TOTP / WebAuthn |

### 🔑 11. 密码管理 — Vaultwarden

| 功能 | 说明 |
|------|------|
| 服务器密码 | 集中加密存储 |
| SSH 密钥 | 统一管理 |
| 分享 | 团队共享密码不传明文 |
| 客户端 | 手机/浏览器/命令行都能用 |

### 📦 12. 配置备份 — restic + MinIO

| 组件 | 用途 |
|------|------|
| restic | 增量备份、加密、去重 |
| MinIO | S3 兼容存储，存备份文件 |
| 策略 | 每天凌晨 3 点自动备份所有配置 |

## 三、分阶段实施计划

### 🟢 第一阶段：核心监控（第 1 周）

| 任务 | 工具 | 部署方式 |
|------|------|---------|
| 自动发现 + 基础监控 | LibreNMS | docker compose |
| 深度指标采集 | Prometheus + Node Exporter | 每台机器装 exporter |
| 统一仪表盘 | Grafana | docker |
| 告警通知 | LibreNMS Alerting → 钉钉 | webhook 配置 |

### 🟡 第二阶段：管理与自动化（第 2 周）

| 任务 | 工具 |
|------|------|
| 资产管理 CMDB | NocoDB |
| 远程管理 | Cockpit |
| 工作流自动化 | n8n |
| 密码管理 | Vaultwarden |

### 🟠 第三阶段：日志与安全（第 3 周）

| 任务 | 工具 |
|------|------|
| 日志集中管理 | Loki + Promtail |
| 漏洞扫描 | OpenVAS |
| 统一认证 | Authentik |
| 报表 | Grafana Reports |

### 🔵 第四阶段：加固与备份（第 4 周）

| 任务 | 工具 |
|------|------|
| 配置备份 | restic + MinIO |
| Nginx 统一入口 + HTTPS | Nginx + Let's Encrypt |
| 操作审计 | Authentik Audit Log |
| 文档 | 运维手册 + 故障处理 SOP |

## 四、部署拓扑

```
物理服务器 / 虚拟机（建议 .171 CentOS）
├── Docker Engine
│   ├── LibreNMS        (端口 8000)
│   ├── Grafana          (端口 3000)
│   ├── Prometheus       (端口 9090)
│   ├── NocoDB           (端口 8081)
│   ├── n8n              (端口 5678)
│   ├── Loki             (端口 3100)
│   ├── Vaultwarden      (端口 8082)
│   ├── Authentik        (端口 9000)
│   ├── MinIO            (端口 9001)
│   └── OpenVAS          (端口 9392)
├── Nginx (反向代理，443)
│   └── Let's Encrypt 证书自动续期
└── restic 定时备份到 MinIO

被管理设备（.177 / .168 / .179）
├── node_exporter (Prometheus 指标)
├── promtail (日志采集)
├── snmpd (SNMP 发现)
└── Cockpit (远程管理)
```
