# 第七章 Linux 软件包的管理与安装

> 来源：Web安全渗透+Kali渗透高级工程师 VIP 课程

---

## 7.1 RPM 软件包管理

### 软件包类型

| 类型 | 说明 | 后缀 |
|------|------|------|
| **RPM 二进制包** | 已用 GCC 编译，可直接执行 | `.rpm` |
| **源码包** | 需手动编译安装 | `.tar.gz` / `.src.rpm` |

> Kali 用的是 `apt install`（deb 包），CentOS 用 `rpm`/`yum`。

### RPM 包命名规则

```
zsh-5.5.1-9.el7.x86_64.rpm
 │   │ │ │  │   └── 64位包
 │   │ │ │  └── 操作系统版本（el7 = RHEL 7.x）
 │   │ │ └── release 发布版本号（第几次编译）
 │   │ └── 修订号（第几次修 bug）
 │   └── 次版本号
 └── 软件名 + 主版本号
```

| 后缀关键词 | 含义 |
|-----------|------|
| `x86_64` | 64 位 |
| `noarch` | 任何平台都可安装 |
| `devel` | 开发包 |
| `.src.rpm` | 源码包 |

### 7.1.1 安装 RPM 包

```bash
rpm -ivh 软件包全名
# -i  install 安装
# -v  显示详细信息
# -h  显示 #### 进度条
```

```bash
mount /dev/sr0 /mnt                               # 挂载光盘
rpm -ivh /mnt/Packages/lrzsz-0.12.20-43.el7.x86_64.rpm
```

> 📌 **全名 vs 包名：**
> - 安装/升级 → 用**全名**
> - 查询/卸载已安装的 → 用**包名**（默认查 `/var/lib/rpm/`）

### 7.1.2 查询 RPM 包

```bash
rpm -q lrzsz                       # 查询是否安装
rpm -qa                            # 查询所有已安装包
rpm -qa | grep vim                 # 按关键词筛选
rpm -qf /usr/bin/find              # 反向查：文件属于哪个包
rpm -qi lrzsz                      # 查看包详细信息
rpm -ql lrzsz                      # 查看包安装的文件列表
rpm -qR lrzsz                      # 查看包的依赖
```

### 7.1.3 卸载和升级

```bash
rpm -e lrzsz                       # 卸载
rpm -e --nodeps lrzsz              # 忽略依赖强制卸载（不推荐）
rpm -Uvh 软件包全名                  # 升级（推荐用 yum upgrade）
```

### 7.1.4 依赖问题

```bash
# 单独装 httpd 会报依赖错误
rpm -ivh httpd-xxx.rpm
# 错误：依赖检测失败

# 解决：用 yum 自动解决依赖
yum -y install /mnt/Packages/httpd-xxx.rpm
```

## 7.2 YUM 包管理器

> YUM（Yellow dog Updater Modified）基于 RPM，**自动解决依赖**，基于 C/S 架构。

### 7.2.1 配置本地 YUM 源

```bash
# 1. 挂载光盘
mount /dev/cdrom /mnt/

# 2. 备份在线源
cd /etc/yum.repos.d/
mv CentOS-Base.repo CentOS-Base.repo.bak

# 3. 配置本地源
vim CentOS-Base.repo
```

```ini
[base]
name=CentOS
baseurl=file:///mnt/
gpgcheck=0
```

```bash
# 4. 重建缓存
yum clean all && yum makecache
```

### 7.2.2 YUM 常用命令

| 命令 | 作用 |
|------|------|
| `yum -y install 包名` | 安装软件包 |
| `yum update` | 更新软件+系统+内核 |
| `yum upgrade` | 只升级软件，不升级内核（推荐） |
| `yum info 包名` | 查看包信息 |
| `yum provides /usr/bin/find` | 查命令属于哪个包 |
| `yum deplist 包名` | 查看依赖 |
| `yum -y remove 包名` | 卸载 |
| `yum search 关键词` | 搜索包 |
| `yum grouplist` | 查看软件包组 |
| `yum groupinstall 'Security tools'` | 安装软件包组 |

### 7.2.3 切换回在线源

```bash
cd /etc/yum.repos.d/
rm -rf CentOS-Base.repo
mv CentOS-Base.repo.bak CentOS-Base.repo
yum clean all && yum makecache
```

## 7.3 源码包安装（实战：编译 Nginx）

### 7.3.1 准备工作

```bash
# 安装编译依赖
yum -y install gcc gcc-c++ make zlib-devel pcre pcre-devel openssl-devel
```

| 依赖包 | 作用 |
|--------|------|
| `gcc` | C 语言编译器 |
| `gcc-c++` | C++ 编译器 |
| `make` | 编译工具 |
| `zlib-devel` | gzip 压缩模块 |
| `pcre-devel` | 正则表达式、URL 重写 |
| `openssl-devel` | SSL 功能 |

### 7.3.2 源码编译三把斧

```bash
# 1. 解压
tar xvf nginx-1.24.0.tar.gz
cd nginx-1.24.0

# 2. 配置（检查环境 + 指定安装路径）
./configure --prefix=/usr/local/nginx
echo $?    # 0 = 成功

# 3. 编译
make -j 4    # -j 4 用 4 核提速

# 4. 安装
make install

# 5. 创建软链接（方便直接执行 nginx 命令）
ln -s /usr/local/nginx/sbin/nginx /usr/bin/nginx
nginx
```

### 7.3.3 三把斧详解

| 步骤 | 作用 |
|------|------|
| `./configure` | 指定安装路径、启用/禁用功能、检查依赖，生成 Makefile |
| `make -j 4` | 编译源码为二进制文件 |
| `make install` | 按 Makefile 安装到指定路径 |
| `make clean` | 清除编译产物（需重新 configure 时先执行） |

### 7.3.4 卸载源码包

```bash
# 直接删除 --prefix 指定的目录即可
rm -rf /usr/local/nginx/
```

> 💡 源码安装时必加 `--prefix`，卸载就是一个 `rm -rf`，干干净净！

### 7.3.5 排错示例

```bash
# 缺少 zlib 库时的报错
./configure --prefix=/usr/local/nginx
# checking for zlib library ... not found
# ./configure: error: the HTTP gzip module requires the zlib library.
echo $?    # 1（非 0 表示失败）

# 修复
yum search zlib
yum -y install zlib-devel
./configure --prefix=/usr/local/nginx
echo $?    # 0
```
