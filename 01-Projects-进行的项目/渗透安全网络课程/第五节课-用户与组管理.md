# 第五章 CentOS 7 用户和组管理

> 来源：Web安全渗透+Kali渗透高级工程师 VIP 课程

---

## 5.1 用户账号

> Linux 通过**用户**和**用户组**实现访问控制——包括对文件访问、设备使用的控制。

### 5.1.1 用户分类

| 角色 | UID 范围 (CentOS 7/8) | UID 范围 (CentOS 6) | 说明 |
|------|----------------------|---------------------|------|
| **超级用户** root | `0` | `0` | 系统最高权限 |
| **系统用户** | `1-999` | `1-499` | 用于运行服务，不登录 |
| **本地用户** | `1000+` | `500+` | 普通登录用户 |
| **虚拟用户** | — | — | 伪用户，维持服务运行（ftp、apache 等） |

**用户与组的关系：**

| 关系 | 说明 |
|------|------|
| 一对一 | 一个用户属于一个组 |
| 一对多 | 一个用户属于多个组（一个主组 + 多个附加组） |
| 多对一 | 多个用户属于一个组 |
| 多对多 | 多个用户属于多个组 |

### 5.1.2 三大配置文件

| 文件 | 说明 |
|------|------|
| `/etc/passwd` | 用户基本信息，所有用户可读 |
| `/etc/group` | 用户组信息，组名不能重复 |
| `/etc/shadow` | 用户密码信息，**只有 root 可读** |

**/etc/passwd 字段解读：**
```
harry:x:1001:1001:描述信息:/home/harry:/bin/bash
 ①   ②  ③   ④     ⑤       ⑥         ⑦
```

| 字段 | 含义 |
|------|------|
| ① 用户名 | 登录名 |
| ② 密码占位符 | `x` 表示密码在 `/etc/shadow` |
| ③ UID | 用户 ID，0-65535 |
| ④ GID | 初始组 ID |
| ⑤ 描述信息 | 用户全名/备注（登录界面显示） |
| ⑥ 家目录 | 用户登录后的默认目录 |
| ⑦ Shell | 登录后使用的 Shell |

## 5.2 用户管理

### 5.2.1 useradd — 创建用户

```bash
useradd [-u UID] [-d 家目录] [-g 初始组] [-G 附加组] [-s Shell] [-M] 用户名
```

| 参数 | 说明 |
|------|------|
| `-u` | 指定 UID |
| `-d` | 指定家目录路径 |
| `-M` | **不创建**家目录 |
| `-g` | 指定**初始组**（只能一个） |
| `-G` | 指定**附加组**（可多个，逗号分隔） |
| `-s` | 指定登录 Shell |

### 5.2.2 基础创建

```bash
useradd harry                        # 创建用户（自动创建同名组）
tail -1 /etc/passwd                  # 验证
# harry:x:1001:1001::/home/harry:/bin/bash
```

### 5.2.3 指定 UID

```bash
useradd -u 1100 oracle
id oracle                           # uid=1100(oracle) gid=1100(oracle)
```

### 5.2.4 指定家目录

```bash
useradd -d /opt/ftp ftp1            # 家目录为 /opt/ftp
```

### 5.2.5 指定主组

```bash
useradd -g xuegod xuegod2           # xuegod2 的初始组为 xuegod
id xuegod2                          # gid=1103(xuegod)
```

### 5.2.6 指定附加组

```bash
useradd -G xuegod,oracle,root xuegod3    # 加入 3 个附加组
id xuegod3
# 组=1104(xuegod3),1100(oracle),1102(xuegod),0(root)
```

### 5.2.7 禁止登录的用户

```bash
useradd -s /sbin/nologin xuegod4    # Shell 设为 nologin，禁止登录
```

> 📌 `adduser` 是 `useradd` 的软链接，完全等价：
> ```bash
> ll /usr/sbin/adduser
> # lrwxrwxrwx ... /usr/sbin/adduser -> useradd
> ```

### 5.2.8 userdel — 删除用户

```bash
userdel -r xuegod3     # -r：同时删除家目录和 /var/mail 邮件
```

> ⚠️ `-r` 慎用！确认用户数据不再需要再删。

### 5.2.9 passwd — 密码管理

**交互式修改密码：**
```bash
passwd oracle
# New password:
# Retype new password:
```

**非交互式（脚本用）：**
```bash
echo 123456 | passwd --stdin xuegod      # 一行命令改密码！
echo 123456 | passwd --stdin harry
```

> 🔴 `/etc/shadow` 存放加密后的密码，只有 root 可读。

**⚠️ 扩展：UID 设为 0 变 root**
```bash
# 编辑 /etc/passwd，把某用户 UID 改为 0
xuegod:x:0:0:xuegod:/home/mk:/bin/bash    # UID=0 → 等同 root 权限！
```

> 🔴 这是提权的基本原理之一，理解就行，别乱改！

### 5.2.10 usermod — 修改用户

```bash
usermod [-u UID] [-d 家目录] [-g 初始组] [-G 附加组] [-s Shell] [-c 描述] 用户名
```

| 操作 | 命令 |
|------|------|
| 改 UID | `usermod -u 1111 oracle` |
| 改家目录 | `usermod -m -d /tmp/aaa xuegod` （`-m` 自动迁移文件） |
| 改描述 | `usermod -c "hello world" xuegod` |
| 改 Shell | `usermod -s /sbin/nologin xuegod` |

> 💡 记不住命令？直接 `vim /etc/passwd` 改也一样！

### 5.2.11 模板文件修复

**症状：** `su -` 切换用户后提示符变成 `-bash-4.2$`（不完整）

**原因：** 用户家目录下的 `.bash*` 模板文件被删了

**修复：**
```bash
cp /etc/skel/.bash* /home/xuegod88/                 # 从模板目录复制
chown xuegod88:xuegod88 /home/xuegod88/.bash*       # 修改所有者
```

### 5.2.12 用户查看命令

| 命令 | 作用 |
|------|------|
| `id` | 查看用户 UID/GID/组 |
| `whoami` | 当前有效用户名 |
| `who` | 当前登录用户列表 |
| `w` | 登录用户及正在执行的操作 |
| `users` | 当前登录用户名列表 |

## 5.3 用户切换 — su

```bash
su 用户名       # 切换用户，不切换工作目录
su - 用户名     # 切换用户，同时切换工作目录（推荐）
```

> 📌 root → 任何用户：**不需要密码**
> 
> 📌 普通用户 → 其他用户：**需要输入目标用户的密码**

## 5.5 文件权限（入门）

```
-    rwx         r-x       r-x     root     root  FILENAME
类型  拥有者(u)权限  属组(g)权限  其他人(o)权限  拥有者   属组    文件名
```

| 权限 | 数字 | 含义 |
|------|------|------|
| `r` | 4 | 读（read） |
| `w` | 2 | 写（write） |
| `x` | 1 | 执行（execute） |
