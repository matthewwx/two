# 第六章 Linux 文件权限管理

> 来源：Web安全渗透+Kali渗透高级工程师 VIP 课程

## 6.1 文件的基本权限（UGO）

### 6.1.1 权限的作用

通过对文件设定权限实现三种访问限制：
- 只允许用户自己访问
- 允许预先指定的用户组访问
- 允许任何用户访问

### 6.1.2 查看权限

```bash
ll /etc/passwd
# -rw-r--r--. 1 root root 2053 9月 19 2017 /etc/passwd
```

**权限位拆解：**
```
-        rw-        r--        r--
类型     拥有者(u)   属组(g)    其他人(o)
```

| 首字符 | 文件类型 |
|--------|----------|
| `-` | 普通文件 |
| `d` | 目录 |
| `l` | 符号链接 |
| `s` | socket 套接口文件（如 mysql.sock） |
| `c` | 字符设备（如 tty0） |
| `b` | 块设备（如 sda、cdrom） |

### 6.1.3 权限说明

**对文件：**

| 权限 | 字母 | 数字 | 含义 |
|------|------|------|------|
| 读 | `r` | 4 | `cat` 查看内容 |
| 写 | `w` | 2 | `vim` `echo` 修改内容 |
| 执行 | `x` | 1 | 运行命令或脚本 |

**对目录：**

| 权限 | 含义 | 对应命令 |
|------|------|----------|
| `r` | 列出目录内容 | `ls` |
| `w` | 创建/删除/移动文件 | `touch` `mkdir` `rm` `mv` `cp` |
| `x` | 进入目录 | `cd` |

> ⚠️ **关键理解：** 要读写目录内文件，不仅需要文件的读写权限，**还需要目录的 x 权限**！

```bash
# user1 创建文件，其他人有读权限
su - user1 && touch user1.txt    # -rw-r--r--
# user2 尝试读取 → 失败！（因为 /home/user1 目录没有 x 权限）
su - user2 && cat /home/user1/user1.txt    # Permission denied
# 给目录加 x 权限后 → 成功
chmod o+x /home/user1
```

### 6.1.4 UGO 模型

| 符号 | 含义 |
|------|------|
| **U**ser | 文件所有者（创建者） |
| **G**roup | 用户组（具有相同特征用户的逻辑集合） |
| **O**ther | 其他用户 |

### 6.1.5 常见权限组合

| 权限 | 含义 |
|------|------|
| `-rwx------` | 仅所有者有完整权限 |
| `-rwxr--r--` | 所有者全部，组和其他只读 |
| `-rw-rw-r-x` | 所有者和组可读写，其他只读和执行 |
| `drwx--x--x` | 目录所有者完整，其他只能进入不能读 |
| `drwx------` | 目录仅所有者可访问（如 `/home/user1`） |

### 6.1.6 更改属主和属组 — chown

```bash
chown user:group filename        # 同时改属主和属组
chown user filename              # 只改属主
chown :group filename            # 只改属组（: 可用 . 代替）
chown -R :group dir              # -R 递归修改目录下所有内容
```

```bash
# 示例
chown user1 a.txt                # 属主 → user1
chown user1:user1 a.txt          # 属主和属组 → user1
chown :root a.txt                # 只改属组 → root
chown -R :yum dir1               # 递归改目录
```

### 6.1.7 修改权限 — chmod（字符法）

```bash
chmod [对谁操作] [操作符] [权限] 文件名
```

| 对谁 | 操作符 | 权限 |
|------|--------|------|
| `u` 所有者 | `+` 添加 | `r` `w` `x` |
| `g` 组 | `-` 减少 | |
| `o` 其他人 | `=` 直接等于 | |
| `a` 所有人 | | |

```bash
chmod u+x,g=-,o-r 1.txt          # 所有者+x，组清空，其他人-r
```

### 6.1.8 权限码 — 数字法

| 权限 | 二进制 | 八进制 |
|------|--------|--------|
| `---` | 000 | 0 |
| `--x` | 001 | 1 |
| `-w-` | 010 | 2 |
| `-wx` | 011 | 3 |
| `r--` | 100 | 4 |
| `r-x` | 101 | 5 |
| `rw-` | 110 | 6 |
| `rwx` | 111 | 7 |

```bash
chmod 755 file    # rwxr-xr-x（所有者全部，组和其他读+执行）
chmod 700 file    # rwx------（仅所有者）
chmod 644 file    # rw-r--r--（所有者读写，其他只读）
chmod 777 file    # rwxrwxrwx（所有人全部权限，⚠️ 不安全！）
```

### 6.1.9 权限对文件和目录的影响总结

| 权限 | 对文件 | 对目录 |
|------|--------|--------|
| `r` | 读取文件内容 | 列出文件名（`ls`） |
| `w` | 修改文件内容 | 创建/删除文件 |
| `x` | 执行文件 | 进入目录（`cd`） |

## 6.2 特殊权限 + ACL

### 6.2.1 SUID / SGID / Sticky

| 特殊权限 | 数字 | 设置方式 | 作用对象 | 功能 |
|----------|------|----------|----------|------|
| **SUID** | 4 | `chmod u+s` / `chmod 4xxx` | 二进制可执行程序 | 运行时以**程序所有者**权限执行 |
| **SGID** | 2 | `chmod g+s` / `chmod 2xxx` | 二进制程序 / 目录 | 目录下新建文件**继承上级目录的组** |
| **Sticky** | 1 | `chmod o+t` / `chmod 1xxx` | 目录 | 只有文件创建者/root/目录所有者能删除 |

**权限位显示：**
```
rwsr-xr-x    SUID（s 在 u 位）
rwxr-sr-x    SGID（s 在 g 位）
rwxrwxrwt    Sticky（t 在 o 位）
rwsr-sr-t    SUID + SGID + Sticky（7755）
```

**SUID 示例（渗透关键！）：**
```bash
# 普通用户无法读 shadow
su - user3 && less /etc/shadow         # Permission denied

# root 给 less 加 SUID
chmod u+s /usr/bin/less                # 或 chmod 4755 /usr/bin/less

# 普通用户再次执行 less → 以 root 身份运行，可读 shadow！
su - user3 && less /etc/shadow         # 成功！

# 查看效果
ll /usr/bin/less                        # -rwsr-xr-x（注意 s）
ps aux | grep less                      # 进程以 root 运行！

# 去掉 SUID
chmod u-s /usr/bin/less
```

> 🔴 **SUID 是 Linux 提权的核心机制之一**，找到有 SUID 位的程序 = 可能的提权路径！

**SGID 示例：**
```bash
mkdir test && chmod g+s test && chown :bin test/
touch test/a.txt
ll test/a.txt    # -rw-r--r-- 1 root bin ...  ← 文件自动继承 bin 组！
```

**Sticky 示例（/tmp 的原理）：**
```bash
ll -d /tmp/    # drwxrwxrwt  ← t = Sticky 位
# 在 /tmp 下：任何人都能创建文件，但只能删自己的
```

### 6.2.2 扩展权限 ACL

> ACL = Access Control List，实现对**特定用户**的精细化权限控制。

```bash
# 查看 ACL
getfacl /opt/a.txt

# 设置 ACL
setfacl -m u:user1:rwx /opt/a.txt            # 给 user1 加 rwx
setfacl -m u:user2:rwx,u:user3:- /opt/a.txt  # user2=rwx, user3=无权限

# 移除 ACL
setfacl -x u:user1 /opt/a.txt                # 移除 user1 的 ACL
setfacl -b /opt/a.txt                        # 移除所有 ACL

# 递归设置
setfacl -R -m u:user1:rw- /opt/test          # -R 必须在 -m 前面
```

> 📌 设置了 ACL 的文件，`ll` 会显示 `+` 号：`-rw-rwxr--+`

## 6.3 实战：创建 root 都无法删除的文件

> 🔥 黑客攻防实用技巧

### chattr — 文件扩展属性

| 参数 | 含义 |
|------|------|
| `+i` | **不可变（Immutable）**，不能修改/删除/重命名 |
| `+a` | **仅追加（Append）**，只能 `>>` 追加内容 |
| `-i` | 移除 i 属性 |
| `-a` | 移除 a 属性 |

```bash
# 设置不可变
chattr +i hack.sh
rm -rf hack.sh         # ❌ 失败！root 也删不掉

# 查看属性
lsattr hack.sh         # ----i-----------

# 解除
chattr -i hack.sh
rm -rf hack.sh         # ✅ 成功

# 仅追加模式
chattr +a hack.sh
rm -rf hack.sh         # ❌ 不能删
echo "data" >> hack.sh # ✅ 可以追加

# 目录也可设 +i
chattr +i dir3
touch dir3/a.txt       # ❌ 不能在目录下新建文件
```
