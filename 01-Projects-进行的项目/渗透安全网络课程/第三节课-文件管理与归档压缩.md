## 3.1 Linux 系统目录结构和相对路径与绝对路径

### 3.1.1 系统目录结构

WIN 是多根（`c:\` `d:\` `e:\`），**Linux 只有一个 `/` 根目录**。

```bash
yum install tree         # 安装 tree 命令
tree /tmp/               # 查看 /tmp 目录结构
ls /                     # 查看根目录
# bin  dev  home  lib64  mnt  proc  run  srv  tmp  var
# boot etc  lib   media  opt  root  sbin  sys  usr
```

**根目录下各目录作用：**

| 目录 | 说明 |
|------|------|
| `/` | Linux 文件系统最顶端，所有目录/文件/设备的入口 |
| `/bin` | Binary 缩写，常用二进制命令（ls、cp、mkdir、cut） |
| `/boot` | 系统启动相关文件（kernel、grub） |
| `/dev` | Device 设备文件（`/dev/sda`、`/dev/tty1`、`/dev/null`、`/dev/cdrom`） |
| `/etc` | 系统和软件配置文件目录（passwd、shadow、group、fstab、hosts） |
| `/home` | 普通用户家目录默认存放位置 |
| `/lib` `/lib64` | 库文件目录，类似 Windows 的 DLL |
| `/mnt` `/media` | 临时挂载存储设备（U盘、光盘等） |
| `/opt` | 可选软件安装目录（如 gitlab） |
| `/proc` | ⚠️ 虚拟目录，系统内存映射，存放进程和内核信息 |
| `/sys` | 系统目录，存放硬件信息 |
| `/run` | 系统运行时数据，如进程 PID 文件 |
| `/srv` | 服务目录，本地服务相关文件 |
| `/sbin` | 系统管理命令（仅 root 可执行） |
| `/tmp` | 临时文件目录（粘滞位 sticky bit 权限，定期清理） |
| `/usr` | 应用程序和文件（`/usr/bin`、`/usr/sbin`、`/usr/lib`） |
| `/var` | 日志等经常变动的文件（`/var/log`、`/var/spool/cron`） |

**库文件知识：**
- `lib***.a` → 静态库，编译时加载到二进制文件
- `lib***.so` → 动态库，运行时加载到进程内存空间

**查看 CPU 信息：** `cat /proc/cpuinfo`

**查看 /tmp 特殊权限：**
```bash
ls -ld /tmp
# drwxrwxrwt ... /tmp/  → 粘滞位（sticky bit），只有文件 owner 和 root 能删除
```

### 3.1.2 绝对路径和相对路径

| 类型 | 定义 | 示例 |
|------|------|------|
| **绝对路径** | 从 `/` 开始的完整路径 | `/etc/passwd`、`/home/user1` |
| **相对路径** | 从当前目录开始的路径 | `a.txt`、`./a.txt`、`../root/b.txt` |

```bash
pwd              # 查看当前位置
cd /etc/         # 切换到 /etc
ll passwd        # 相对路径
ll /etc/passwd   # 绝对路径（效果一样）
```

## 3.2 文件的管理

### 命令速查

| 命令 | 作用 | 常用选项 |
|------|------|----------|
| `touch` | 创建空文件 / 修改文件时间 | — |
| `mkdir` | 创建目录 | `-p` 递归创建 |
| `vim` | 创建/编辑文件 | — |
| `echo >` | 重定向创建文件 | — |
| `rm` | 删除文件或目录 | `-f` 强制，`-r` 递归 |
| `cp` | 复制文件或目录 | `-r` 递归复制目录 |
| `mv` | 移动/重命名文件 | — |

### 3.2.1 创建文件和文件夹 — touch

```bash
touch a.txt                  # 创建空文件
touch file1 file2            # 同时创建多个
touch file{6..20}            # 批量创建 file6~file20（共15个）

# 如果文件已存在，touch 会更新文件的修改时间
```

> ⚠️ **Linux 不以扩展名判断文件类型，严格区分大小写！**

### 3.2.2 用 vim 和重定向创建文件

```bash
vim xuegod-1.txt            # vim 创建
echo aaa > xuegod.txt        # 重定向创建（> 覆盖，>> 追加）
```

### 3.2.3 创建目录 — mkdir

```bash
mkdir dir1                          # 创建单个目录
mkdir dir2 dir3 /home/dir4          # 同时创建多个
mkdir -p /tmp/a/b/c                 # -p 递归创建（上级不存在时自动创建）
```

### 3.2.4 删除 — rm

```bash
rm -rf a.txt           # 强制删除文件
rm -rf dir             # 强制删除目录（递归）
rm -rf file*           # 通配符批量删除
```

> 🔴 **`rm -rf` 慎用！删除前一定确认所在目录，防止误删重要数据！**

### 3.2.5 复制 — cp

```bash
cp /etc/passwd /opt/                     # 复制文件
cp -r /boot/grub2 /opt/                  # -r 递归复制目录
cp -r /boot/grub2 /etc/passwd /opt       # 复制多个文件到目标目录
\cp -r /boot/grub2 /opt                  # \ 反斜杠：覆盖不提示
```

### 3.2.6 移动/重命名 — mv

```bash
mv passwd bbbpasswd              # 重命名
mv bbbpasswd /opt/passwd         # 移动并改名
```

## 3.3 查看文件

### 命令对比

| 命令 | 特点 | 适用场景 |
|------|------|----------|
| `cat` | 一次性显示全部内容 | 小文件 |
| `more` | 分页显示，不支持后退 | 快速浏览 |
| `less` | 分页显示，**支持前后翻页** | 大文件阅读 |
| `head` | 查看文件**开头**（默认10行） | 看文件头 |
| `tail` | 查看文件**尾部**（默认10行） | 看日志 |

### 3.3.1 cat

```bash
cat /etc/passwd         # 一次性显示全部
```

### 3.3.2 more

```bash
more /proc/cpuinfo      # 分页显示
# 操作：回车=下一行，空格=下一屏，q=退出
```

### 3.3.3 less

```bash
less /proc/cpuinfo
# 操作：支持上下翻页（PageUp/PageDown），q=退出
```

> **more vs less：** more 不支持后退；less 支持前后翻滚，功能更强。

### 3.3.4 head

```bash
head /etc/passwd         # 默认显示前10行
head -n 3 /etc/passwd    # 显示前3行
```

### 3.3.5 tail

```bash
tail -n 3 /var/log/secure    # 显示最后3行
tail -3 /var/log/secure      # 同上，简写
tail -f /var/log/secure      # 🔥 动态跟踪日志（最常用！）
```

**tail -f 实战示例：**
```bash
# 终端1：动态监控安全日志
tail -f /var/log/secure

# 终端2：SSH 远程登录
ssh root@192.168.1.63

# 终端1 会实时显示：
# Nov 17 00:08:32 xuegod63 sshd[2924]: Accepted password for root from 192.168.1.63 port 39904 ssh2
```

## 3.4 tar 命令 — 归档和压缩

> 好处：节约硬盘资源，加快文件传输速率。

### 3.4.1 归档（打包，不压缩）

**tar 参数速查：**

| 参数 | 说明 |
|------|------|
| `-c` | create 创建归档 |
| `-x` | extract 解压还原 |
| `-v` | verbose 显示详细过程 |
| `-f` | file 指定归档文件名 |
| `-t` | list 列出包中内容（不解包） |
| `-C` | directory 指定解压位置（大写） |

```bash
# 打包
tar -cvf grub.tar /boot/grub2/

# 查看包内容（不解包）
tar -tvf grub.tar

# 解包
tar -xvf grub.tar
tar -xvf grub.tar -C /opt/         # 解压到指定目录
```

### 3.4.2 归档 + 压缩

**三种压缩方式对比：**

| 参数 | 压缩方式 | 扩展名 | 压缩比 | 速度 |
|------|----------|--------|--------|------|
| `-z` | gzip | `.tar.gz` | 中（9.8M） | 快 |
| `-j` | bzip2 | `.tar.bz2` | 较高（8.7M） | 中 |
| `-J` | xz | `.tar.xz` | **最高（7.0M）** | 最慢 |

```bash
# 压缩
tar czvf etc.tar.gz /etc          # gzip 压缩
tar cjvf etc.tar.bz2 /etc         # bzip2 压缩
tar cJvf etc.tar.xz /etc          # xz 压缩

# 解压
tar xzvf etc.tar.gz               # gzip
tar xjvf etc.tar.bz2              # bzip2
tar xJvf etc.tar.xz               # xz

# 通用解压（tar 自动识别压缩类型）
tar xvf etc.tar.gz -C /opt        # 推荐！-C 指定解压目录
```

### 3.4.3 高级用法

```bash
# 解压指定文件到指定目录
tar -C /opt -xvf boot.tar.gz boot/grub2/grub.cfg

# 解压某一类型文件（通配符）
tar -C /opt -xvf boot.tar.gz --wildcards *.lst

# 排除指定文件后解压
tar -C /opt -xvf boot.tar.gz boot/grub2 --exclude=boot/grub2/i386-pc
```

## 3.5 zip 管理压缩文件

```bash
# 压缩
zip a.zip /etc/passwd             # 压缩单个文件
zip all.zip *.jpg                 # 通配符压缩

# 解压
unzip grub.zip                    # 解压到当前目录
unzip grub.zip -d /opt/           # -d 指定解压目录
```

> zip/unzip 跨平台兼容性好（Linux ↔ Windows），但压缩效率不如 tar.gz。
