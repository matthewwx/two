# 新人小白自学挖漏洞路线

> 来源：一号提供的 PDF 文件，整理为结构化学习计划

![[../03-Resources-参考资料及收藏的资源/附件/新人小白挖漏洞路线图.png]]

## 核心理念

> 💡 **没学过英语、没学过编程，也能学会挖漏洞**
> 
> SRC 漏洞中 **70% 是逻辑漏洞**，不需要深入编程，靠抓包 + 逻辑分析就能挖到

---

## 第一阶段：基础知识（1-2 周）

### 1.1 什么是漏洞

在网站/系统里找到安全缺陷，能够绕过限制、获取越权数据、拿到不该拿的信息→提交给厂商换赏金。

### 1.2 装好环境

| 工具         | 用途            |
| ---------- | ------------- |
| Kali Linux | 渗透专用系统（先装虚拟机） |
| Burp Suite | ⭐ 核心抓包工具 — [[06-wiki/Burp Suite使用教程\|教程]] |
| 浏览器 F12    | 前端调试          |

### 1.3 Burp Suite 核心功能（重点学这个）

| 功能 | 说明 | 重要性 |
|------|------|--------|
| **Repeater** | 重放请求，修改参数反复测试 | ⭐⭐⭐ 最常用 |
| **Intruder** | 爆破/遍历参数 | ⭐⭐⭐ |
| **Decoder** | 编解码（URL、Base64、Hex、Unicode） | ⭐⭐ |
| 推荐插件 | Logger++、Hae | ⭐⭐ |

> 小迪网站有 BP 教程，跟着学。

## 第二阶段：HTTP 协议（1 周，重点）

### 2.1 网络基础

- TCP 三次握手
- HTTP/HTTPS 请求响应流程

### 2.2 请求方法

| 方法 | 说明 |
|------|------|
| GET | 获取数据，参数在 URL |
| POST | 提交数据，参数在 body |
| PUT / DELETE / OPTIONS / HEAD | 了解即可 |

### 2.3 关键请求头

```
Host、Cookie、User-Agent、Referer、Content-Type、Authorization
```

### 2.4 响应状态码（重点防 200/403 误判）

| 状态码 | 含义 |
|--------|------|
| 200 | 成功 |
| 301/302 | 跳转 |
| 400 | 请求错误 |
| 401 | 未授权 |
| 403 | 禁止访问 |
| 404 | 不存在 |

### 2.5 数据格式

`form-data` / `x-www-form-urlencoded` / `JSON` / `XML`

| 概念 | 说明 |
|------|------|

## 第四阶段：漏洞挖掘核心思路（核心重点）

### 4.1 盯住这些参数

```
orderid=    → 改数字看别人订单
id=         → 改数字看别人数据
userid=     → 改数字切换用户身份
```

### 4.2 重点漏洞类型

| 漏洞类型 | 说明 | 难度 |
|----------|------|------|
| **越权（IDOR）** | ⭐ 核心中的核心，改 ID 看别人的数据 | 低 |
| **支付漏洞** | 改金额、改数量、改优惠券 | 低-中 |
| **验证码爆破** | 四位/六位验证码可遍历 | 低 |
| **任意用户登录** | 篡改返回包、Cookie 伪造 | 中 |

### 4.3 实操方法

1. 打开 Burp Suite 挂上代理
2. 正常操作网站（登录、下单、查数据）
3. 看每个请求里有哪些参数
4. 改参数值 → Repeater 重放 → 看返回结果
5. 状态码 200 + 数据变 → 漏洞！

### 5.1 怎么学

- 公众号、知乎、安全论坛看最新漏洞思路

- 以实战为学习

## 学习节奏建议

| 周次 | 内容 | 目标 |
|------|------|------|
| 第1周 | 装 Kali + Burp Suite，学会抓包 | 能用 BP 拦截和重放请求 |
