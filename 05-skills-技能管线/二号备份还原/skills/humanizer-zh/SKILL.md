# Humanizer-zh — 中文去 AI 味

> 24 条规则消灭 AI 写作痕迹，让文字更像「人话」

## 使用方式

输入 `/humanizer-zh` 后跟上要处理的文本，我会按规则检测并改写。

---

## 四大类 AI 痕迹检测规则

### 一、内容模式（6 条）

1. **过度强调意义** — 删除"在当今快速发展的 X 格局中""已成为至关重要的组成部分"等空洞拔高
2. **宣传语言** — 删除"无缝""高效""可扩展""革命性"等营销套话
3. **模糊归因** — "研究表明""数据显示"但没说哪个研究、什么数据 → 要么补充来源，要么删除
4. **肤浅分析** — 只列优点不说缺点、只说是什么不说为什么 → 补充具体案例或数据
5. **通用积极结论** — "总之，X 将在未来发挥越来越重要的作用" → 换成具体、有信息量的结尾
6. **AI 式结构** — "首先...其次...最后...总而言之" 的八股框架 → 用更自然的过渡

### 二、语言语法模式（6 条）

1. **AI 高频词汇** — "此外""至关重要""深入探讨""不可忽视""值得注意的是" → 删或换口语表达
2. **三段式法则** — 每个观点都恰好三个论据 → 不强迫三等分
3. **否定式排比** — "不是...不是...而是..." 的万能句式 → 精简
4. **"通过...实现..."** — 翻译腔句式 → 改成中文习惯表达
5. **名词化泛滥** — "对 X 进行 Y" → "Y X"；"起到了 Z 的作用" → 直接用动词
6. **被动语态过度** — "被广泛认为""被普遍使用" → 改成主动态

### 三、写作风格（6 条）

1. **破折号/粗体滥用** — 英文 em-dash 习惯搬进中文 → 用逗号、句号替代
2. **表情符号过多** — 🎯💡🚀 等 → 只保留必要的
3. **弯引号/英文标点** — ""'' 改成中文「」""，逗号句号统一全角
4. **内联标题列表** — 一段话里塞满 `**粗体标题**：解释` → 拆成自然段落
5. **英文字母大小写** — 中文上下文中的英文专有名词合理使用，不要全大写
6. **段落长度均匀** — 每段恰好 3-5 行 → 长短错落

### 四、交流模式（6 条）

1. **谄媚语气** — "您说得太对了！""这是个非常好的问题！" → 去掉或改为中性回应
2. **知识截止免责** — "根据我的训练数据...""截至 202X 年..." → 删除
3. **填充短语** — "我想强调的是""需要指出的是" → 直接说内容
4. **协作痕迹** — "让我们一起来探索...""我们可以..." → 直接陈述
5. **过度解释** — 一个简单概念用三句话反复说 → 一句说清楚
6. **"当然"+ 转折** — "当然，这并不意味着..." → 直接转折

## 改写示例

**原文（AI 味）：**
> 在当今快速发展的安全领域，Burp Suite 已成为渗透测试中至关重要的工具。此外，其丰富的插件生态系统为安全研究人员提供了无缝、高效和可扩展的测试体验——确保团队能够更加敏捷地发现潜在漏洞。

**改写后：**
> Burp Suite 是渗透测试的核心工具。插件市场里有几百个扩展，从自动越权检测到 SQL 注入扫描，大部分常见漏洞都能覆盖。

## 使用原则

- **仅改写**：不改内容含义，只改表达方式
- **保持作者语气**：不要统一改成同一风格
- **非破坏性**：技术术语、代码、数据不碰
- **可拒绝**：原文已经是自然中文 → 告知无需修改
