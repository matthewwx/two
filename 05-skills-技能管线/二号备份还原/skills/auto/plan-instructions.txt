## Your Task

1. Read the current slice from <state> (current_slice field).
2. Find the slice description in <roadmap>.
3. Research the codebase: scan existing files, understand the project structure, check what's already built.
4. Decompose the slice into 1-7 tasks, each fitting one context window.

## For Each Task, Write:

Use this XML format in each task plan file:

```xml
<task id="S{nn}-T{nn}" type="auto">
  <name>{descriptive name}</name>
  <files>{list of files to create/modify}</files>
  <risk level="{low|medium|high}">
    {why this task has this risk level}
  </risk>
  <acceptance_criteria>
    <ac id="AC-{n}">
      Given {precondition}
      When {action}
      Then {expected outcome}
    </ac>
  </acceptance_criteria>
  <action>
    1. {concrete step}
    2. {concrete step}
  </action>
  <boundaries>
    DO NOT CHANGE:
    - {file} ({reason})
  </boundaries>
  <verify>{command} (AC-{n})</verify>
  <done>{completion criteria}</done>
</task>
```

## Rules

- Every task MUST have at least 1 AC in Given/When/Then format.
- Every task MUST have risk level low, medium, or high with a clear reason.
- Every task MUST have a boundaries section.
- No "TBD", "TODO", or "later" in action, files, or risk.
- Files from earlier tasks listed as DO NOT CHANGE in later tasks.
- Tasks ordered by dependency.

## Files to Write

1. `.gsd/S{nn}-PLAN.md` — slice overview with task table including risk, AC table, boundaries summary, dependency order.
2. `.gsd/S{nn}-T01-PLAN.xml` through `.gsd/S{nn}-T{last}-PLAN.xml` — one file per task.
3. `.gsd/M{n}-RESEARCH.md` — append research findings.

Do NOT create `.gsd/S{nn}-T{nn}-PLAN.md`. Markdown is reserved for the slice
plan only.

## Git

Use `base_branch` from <state>. Auto-mode prepares the slice branch from that
base before this prompt runs. Verify the current branch is `gsd/M{n}/S{nn}`.

If the branch was not prepared yet, create it from the configured base branch:

```bash
git switch {base_branch}
git switch -c gsd/M{n}/S{nn}
```

If the slice branch already exists, switch to it instead of recreating it.
Do not switch branches when unrelated worktree changes are present.

After writing the planning artifacts, stage only the slice-scoped files you
changed for this run:
```bash
git add {specific slice planning files} .gsd/M{n}-RESEARCH.md .gsd/STATE.md
```

Then create a planning commit with a slice-specific message:
```bash
git commit -m "docs(S{nn}): plan slice" \
  -m "Auto-mode saved the slice planning artifacts and only staged files owned by this slice."
```

Do NOT use `git add .`, `git add -A`, or repo-wide staging.
